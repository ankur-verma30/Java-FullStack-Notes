ğŸ”· Hibernate Cache
â¤ Hibernate cache = temporary in-memory storage used to reduce database round trips.
â¤ It stores objects/entities so Hibernate doesnâ€™t repeatedly query the DB for the same data.
â¤ Goal = performance improvement â†’ less SQL, lower latency, reduced DB load.

1ï¸âƒ£ First-Level Cache (L1 Cache)
â¤ Enabled by default â€“ cannot be disabled.
â¤ Scope: per Session object (lives as long as the Hibernate Session exists).
â¤ Lifecycle: Cache is cleared once session is closed.
â¤ Data locality: Same session â†’ Hibernate wonâ€™t hit DB twice for the same entity.

âœ…Example:

User user1 = session.get(User.class, 1L);  // ğŸ”´ Hits DB
User user2 = session.get(User.class, 1L);  // âœ… Served from L1 cache, no DB call

session.close();

â“ What happens here?
â¤ First call â†’ Hibernate queries DB (SELECT * FROM user WHERE id=1).
â¤ Second call â†’ Hibernate checks L1 cache (inside the session), finds user1, and reuses it.

âš¡ Important: If you open a new session, cache is cleared â†’ it will hit DB again.

2ï¸âƒ£ Second-Level Cache (L2 Cache)
â¤ Optional (must be enabled via configuration).
â¤ Scope: Shared across multiple Sessions (SessionFactory level).
â¤ Persists cached entities across different users and requests.
â¤ Works with external caching providers (Ehcache, Infinispan, Caffeine, Redis).

âœ…Example:
//Session 1
Session session1 = sessionFactory.openSession();
User user1 = session1.get(User.class, 1L);  // ğŸ”´ Hits DB, stores in L2 cache
session1.close();

// Session 2
Session session2 = sessionFactory.openSession();
User user2 = session2.get(User.class, 1L);  // âœ… Served from L2 cache (no DB hit)
session2.close();

â“ What happens here?
â¤ First session queries DB and puts data in L2 cache.
â¤ Second session â†’ Hibernate checks L2 cache, finds entity, and skips DB.

ğŸ“Š Key Differences: L1 vs L2 Cache
| Feature       | First-Level Cache (L1)                    | Second-Level Cache (L2)                                   |
| ------------- | ----------------------------------------- | --------------------------------------------------------- |
| Default       | Always enabled                            | Optional (must configure)                                 |
| Scope         | Per Session                               | Shared across Sessions                                    |
| Lifetime      | Ends when Session closes                  | Ends when SessionFactory closes                           |
| Use Case      | Prevent duplicate queries in same request | Prevent queries across multiple requests/users            |
| Providers     | Built-in Hibernate                        | Requires providers (Ehcache, Infinispan, Redis, Caffeine) |
