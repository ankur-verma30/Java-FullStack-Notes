üß† Why SQL?
We need a language to interact with databases‚Äîwhether it's to store, retrieve, update, or delete data.
SQL (Structured Query Language) is the standard language used for interacting with Relational Databases.

With SQL, we can perform:

‚û§ Create ‚Äì Create new records or database structures
‚û§ Read ‚Äì Retrieve data from tables
‚û§ Update ‚Äì Modify existing records
‚û§ Delete ‚Äì Remove data
These operations are known as CRUD operations and are essential for any database-driven application.

Key Features and Advantages of Using SQL

‚úîÔ∏è Ubiquity: SQL is a widely used language, making it easy to learn and use.
‚úîÔ∏è Flexibility: SQL supports a wide range of database operations, including data definition, data manipulation, and data control.

‚úîÔ∏è Data Integrity: SQL ensures data integrity and security, making it suitable for enterprise-level applications.

üß† How SQL Helps Us
SQL (Structured Query Language) enables users to interact with relational databases effectively. It supports a wide range of operations, grouped into the following categories:

‚úÖ 1. Retrieving Data
‚û§ SQL allows users to fetch specific and meaningful data from large datasets.
‚û§ Users can apply filters, conditions, sorting, and grouping to extract exactly what is needed.
‚û§ This is essential for generating reports, dashboards, and data analysis.

‚úÖ 2. Manipulating Data
‚û§ SQL provides commands to add, update, and delete records in the database.
‚û§ These operations help maintain up-to-date and accurate information.
‚û§ It supports bulk updates and conditional changes as well.

‚úÖ 3. Defining Data
‚û§ SQL allows the creation and modification of the database structure.
‚û§ This includes creating or altering tables, views, indexes, and constraints.
‚û§ It ensures the data is stored in an organized and consistent format.

‚úÖ 4. Controlling Data
‚û§ SQL supports user access control by allowing database administrators to grant or revoke permissions.
‚û§ It ensures that sensitive data is only accessible to authorized users.
‚û§ Helps enforce security, privacy, and role-based access.

üß† Types of SQL Commands
SQL commands are categorized based on the type of operation they perform on the database:

‚úÖ 1. Data Query Language (DQL)
‚û§ Purpose: To retrieve data from the database.

Command: SELECT
Note: Although it has only one command, it's one of the most frequently used.

‚úÖ 2. Data Manipulation Language (DML)
‚û§ Purpose: To manipulate data stored in existing tables.
Common Operations:
1. Insert new records
2. Update existing records
3. Delete records

Commands: INSERT, UPDATE, DELETE
Affects: Only the data, not the structure.

‚úÖ 3. Data Definition Language (DDL)
‚û§ Purpose: To define or modify the structure of database objects.

Used for:
Creating tables, indexes, and views
Modifying table structure
Deleting database objects

Commands: CREATE, ALTER, DROP, TRUNCATE, RENAME
Note: DDL commands are usually auto-committed (changes are permanent).

‚úÖ 4. Data Control Language (DCL)
‚û§ Purpose: To control access to data and database operations.

Used for:
Granting permissions to users
Revoking access when needed

Commands: GRANT, REVOKE
Helps enforce: Data security and role-based access control.

‚úÖ 5. Transaction Control Language (TCL)
‚û§ Purpose: To manage transactions in a database.

Used to:
Commit changes
Roll back changes in case of errors
Ensure consistency of operations

Commands: COMMIT, ROLLBACK, SAVEPOINT

üß† Understanding Rows and Columns in a Table
Columns = Attributes / Fields / Characteristics
Each column in a table represents a specific attribute of the entity.
Example: In a Student table, columns could be RollNo, Name, Age, Department.

Rows = Records / Tuples
Each row stores a complete set of data for one instance of the entity.
In other words, a row contains all the attribute values for a single entity.

üß† Hierarchy in a Relational Database System

‚úÖ 1. Server
‚û§ A database server hosts and manages one or more databases.
‚û§ It handles connections, storage, and access control.

‚úÖ 2. Database
‚û§ A database is a logical collection of related data.
‚û§ You can create multiple databases on a single server.
‚û§ Each database is independent and contains its own tables, users, permissions, etc.

‚úÖ 3. Tables
‚û§ A database is made up of multiple tables.
‚û§ Each table stores data related to a specific entity (e.g., students, employees).
‚û§ Tables are structured in rows and columns.

‚úÖ 4. Records (Rows)
‚û§ A record (also called a tuple) represents one entry in the table.
‚û§ Each row holds data for one instance of the entity.

‚úÖ 5. Attributes (Columns)
‚û§ Each record is made up of attributes (columns).
‚û§ Attributes represent the properties or characteristics of the entity.
Example: For a Student record, attributes could be Name, Roll No, Marks, etc.

üß† Creating a Table
‚û§ We use the CREATE TABLE statement to define a new table within a database.
‚û§ It is a DDL command.
‚û§ A table consists of columns, each with a data type and optional constraints to enforce rules on the data.

‚úÖ Syntax:

    column1 data_type1 constraint1,
    column2 data_type2 constraint2,
    column3 data_type3 constraint3,
    ...
    table_constraints

‚úÖExample:

üìù Explanation:

‚û§ data_type: Defines the type of data (e.g., INT, VARCHAR, DATE).
‚û§ constraint (optional): Rules like PRIMARY KEY, NOT NULL, UNIQUE, etc.

üß† Data Types in SQL
SQL provides various data types to define the type of data that can be stored in each column of a table.

‚úÖ 1. Numeric Data Types
Used to store numbers (integers or decimals).

‚û§ INT / INTEGER ‚Äì Whole numbers (INT_MIN to INT_MAX)- 4 Bytes
‚û§ SMALLINT ‚Äì Smaller range of integers 2 Bytes
‚û§ TINYINT ‚Äì Smallest range of integers 1 Byte
‚û§ MEDIUMINT ‚Äì Medium range of integers 3 Bytes
‚û§ BIGINT ‚Äì Larger range of integers 8 Bytes

‚û§ DECIMAL(p,s) ‚Äì Fixed-point number with precision p and scale s (e.g., DECIMAL(5,2) ‚Üí 123.45)- Variable Size limit of (p,s) is (65,30).
‚û§ DOUBLE ‚Äì Double-precision floating-point numbers (8-byte) (10,0) as precision and scale
‚û§ FLOAT ‚Äì Approximate floating-point numbers (4-byte) (not always precise)

By default all the numeric dataypes can have negative as well as positive values. This restrict the range so if we know there is only +ve values which is stored we use UNSIGNED attribute (0-255).
 
‚úÖ 2. Character/String Data Types
Used to store text.

‚û§ CHAR(n) ‚Äì Fixed-length string range(0-255) (e.g., CHAR(10) stores exactly 10 characters)
‚û§ VARCHAR(n) ‚Äì Variable-length string range(65535 bytes) (up to n characters)
‚û§ TEXT ‚Äì Large text blocks (no fixed limit like VARCHAR)

‚úÖ 3. Date and Time Data Types

‚û§ DATE ‚Äì Stores a date (YYYY-MM-DD)
‚û§ TIME ‚Äì Stores a time (HH:MM:SS)
‚û§ DATETIME ‚Äì Stores both date and time
‚û§ TIMESTAMP ‚Äì Similar to DATETIME, often used for auto-recording date/time of entry

‚úÖ 4. Boolean Data Type
Used for true/false values.
BOOLEAN / BOOL ‚Äì Stores TRUE or FALSE (internally stored as 1 or 0)

‚úÖ 5. Binary Data Types
Used to store binary data (e.g., images, files).
‚û§ BINARY(n) ‚Äì Fixed-length binary data 255 bytes
‚û§ VARBINARY(n) ‚Äì Variable-length binary data 64KB
‚û§ BLOB ‚Äì Binary Large Object, used to store large binary files like images or documents
Categories of BLOB data types:
    ‚û§ TINYBLOB (255 bytes)

    ‚û§ MEDIUMBLOB (16MB)

‚úÖExample: ENUM('active', 'inactive', 'deleted');

‚û§ Used to store a set of values.

üß† Keys in SQL
Keys are crucial in SQL to uniquely identify records and establish relationships between tables.

‚úÖ 1. Primary Key
‚û§ A Primary Key uniquely identifies each record in a table.
‚û§ It ensures that no two rows have the same value for this key.

A primary key column must be:
1. Unique
2. Not Null

A table can have only one primary key, which can consist of one or more columns (composite key).

üìå Purpose: Uniquely identify each row.

‚úÖ 2. Foreign Key
‚û§ A Foreign Key is a column (or set of columns) in one table that refers to the Primary Key in another table.
‚û§ It is used to establish a relationship between two tables (called referential integrity).
‚û§ A foreign key can have duplicate and null values, unlike primary keys.

üìå Terms:
‚û§ Referenced Table: The table containing the primary key.
‚û§ Referencing Table: The table containing the foreign key.

üìù Example Use Case:
Students table has student_id as primary key.
Enrollments table has student_id as foreign key, referencing the Students table.

üìä Sample Data:
üîπ Students Table:
| student\_id | name  | department |
| ----------- | ----- | ---------- |
| 101         | Ankur | CS         |
| 102         | Riya  | IT         |

üîπ Enrollments Table:
| enrollment\_id | student\_id | course\_name |
| -------------- | ----------- | ------------ |
| 1              | 101         | DBMS         |
| 2              | 102         | SQL          |
| 3              | 101         | Data Science |

The Enrollments.student_id refers to Students.student_id, ensuring relational integrity.

üß† Constraints in SQL
Constraints are rules applied to columns or tables to maintain the accuracy, validity, and integrity of data in a database.

‚úÖ Common SQL Constraints:
üß† SQL Constraints with Examples
Each of the following examples uses a new table to demonstrate a specific constraint.

‚úÖ 1. UNIQUE Constraint
Ensures that all values in a column are distinct (no duplicates).

CREATE TABLE student (
    phonenbr INT UNIQUE
);
-- Inserts unique phone numbers

‚úÖ 2. NOT NULL Constraint
Ensures a column cannot be left empty (NULL values not allowed).

CREATE TABLE student1 (
    age INT,

);
-- Valid: rollno is NOT NULL

‚úÖ 3. CHECK Constraint
Applies a condition that each value in the column must satisfy.

CREATE TABLE student2 (
    age INT CHECK(age > 18)
);
-- Valid: 19 > 18
INSERT INTO student2 VALUES (19);

‚úÖ 4. DEFAULT Constraint
Automatically assigns a default value to a column if no value is specified.

CREATE TABLE student3 (
    schoolName VARCHAR(50) DEFAULT 'RLB',
    age INT
);
-- Only age is given, schoolName will be set to default 'RLB'
INSERT INTO student3 (age) VALUES (23);

-- View inserted data

‚úÖ 5. PRIMARY KEY Constraint
Uniquely identifies each row. It combines UNIQUE + NOT NULL. Only one per table.

CREATE TABLE student4 (
    age INT,
    rollNo INT PRIMARY KEY
);
-- All rollNo values must be unique and not null
INSERT INTO student4 VALUES (12,1), (13,2), (14,3), (15,5), (16,6);

Establishes a relationship between two tables by referencing the primary key of another table.

);
-- Inserting data with rollNo values that exist in student4

üß† Foreign Key in SQL
‚û§ A foreign key creates a relationship between two tables.
‚û§ It ensures that the value in the child table exists in the parent (referenced) table.

‚úÖ Syntax:
CREATE TABLE childTableName (
    childId INT PRIMARY KEY,
    baseId INT,
    FOREIGN KEY (baseId) REFERENCES baseTableName(baseId)
);
üîç Purpose:
‚û§ Enforces referential integrity
‚û§ Helps in JOIN operations between tables

üåä Cascading in Foreign Keys
Cascading options define how changes in the parent table affect the child table.

‚úÖ 1. CASCADE
If a row in the parent is updated/deleted ‚Üí same action is done on the child.
Syntax:
FOREIGN KEY (baseId) REFERENCES baseTable(baseId)
    ON DELETE CASCADE
    ON UPDATE CASCADE

‚úÖ 2. SET NULL
If parent row is updated/deleted ‚Üí sets foreign key in child table to NULL.
Syntax:
FOREIGN KEY (baseId) REFERENCES baseTable(baseId)
    ON DELETE SET NULL
    ON UPDATE SET NULL

‚úÖ 3. RESTRICT / NO ACTION
Prevents parent row from being deleted/updated if it has referencing rows in the child.
Syntax:
FOREIGN KEY (baseId) REFERENCES baseTable(baseId)
    ON DELETE RESTRICT
    ON UPDATE RESTRICT

üß™ Example:
üîπ Parent Table
CREATE TABLE baseTable (
    baseId INT PRIMARY KEY,
    baseName VARCHAR(50)
);

INSERT INTO baseTable VALUES (1, 'Alpha'), (2, 'Beta');

üîπ Child Table with CASCADE
CREATE TABLE childTable (
    childId INT PRIMARY KEY,
    baseId INT,
    FOREIGN KEY (baseId) REFERENCES baseTable(baseId)
        ON DELETE CASCADE
        ON UPDATE CASCADE
);

INSERT INTO childTable VALUES (101, 1), (102, 2);

üîπ Deleting a row in baseTable:
DELETE FROM baseTable WHERE baseId = 1;

‚ö†Ô∏è This will also delete the row in childTable where baseId = 1 because of ON DELETE CASCADE.
