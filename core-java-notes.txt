What is a class?
-> A class in Java is a blueprint or template for creating objects.
-> It defines the data (fields/attributes) and the behavior (methods/functions) that the objects created from the class will have.
-> Class helps us to create objects in Java.

Class Naming Convention
#######################
-> In Java, we use the class keyword to create a class, and the class keyword should be in lowercase. The class name should start with an uppercase letter only.
-> If more than one word is used, then we should use camel casing.

Note: class name cannot start with a number but can begin with $ or underscore(_), but it is not recommended.

Example :
class A {}
-> We define the boundary of a class using curly brackets.
  
Example :
-> Will execute without any error
public class Bank_Account_Number {}// Snake Casing-->Should not use for Java

Note:
1. Do not use specials characters like #,@ % & etc while creating class.
2. $Class name will execute. However, never create a class with a special character ($).

new keyword:
##############
The new keyword in Java is used to create objects (instances) of a class. 
-> Using the new keyword, we send a request to the class to create an object
Once the object is created, the new keyword will retrieve the object's address and store it in a reference variable.

Syntax to create an object:
-------------------------
ClassName variableName = new ClassName();

Two types of camel casing:
a. Upper Case Camel Casing: Used for naming class, interface, abstract class, etc

ðŸ§± Stack Memory â€“ LIFO (Last In First Out)
###########################################
Stores:
âž¤Method calls,
âž¤local variables,
âž¤reference variables (not actual objects).

âž¤Memory is automatically managed (pushed and popped as methods are called or returned).
âž¤Operates in LIFO (Last-In, First-Out) order, where the last is the first one to return.
âž¤Faster but limited in size.

ðŸ§  Example:
When a method is called, its variables go on the stack. When it returns, theyâ€™re removed.

ðŸ—ƒï¸ Heap Memory
################
âž¤Used to store objects/instance variables.
âž¤Every time you do new ClassName(), it creates an object in the heap.
âž¤Objects stay in memory until they're no longer referenced.

ðŸ“ Note: All reference variables (in the stack) point to objects in the heap.

ðŸ—‘ï¸ Garbage Collector (GC)
##########################
Javaâ€™s automatic memory management tool.
Runs in the background and:
1. Detects objects in the heap no longer referenced
2. Removes them to free memory
3. Helps prevent memory leaks and overflow

ðŸ“Œ You donâ€™t have to manually delete objects â€” GC handles it.

---------------------------------------------------------
The Diamond Problem occurs in multiple inheritance when a class inherits from two classes that have a common superclass, leading to ambiguity about which method to inherit.

Suppose we inherit a method from A->B->D,  and the Same method is inherited from A->C->D, then confusion arises from which parent class method is inherited by the child class D. This is called as DIAMOND PROBLEM. Hence, in Java, classes do not support multiple inheritance.

Example: A Database interface may define connect(), disconnect(), but not how these are implemented.

2. Contract Enforcement
âž¤When a class implements an interface, it must implement all its methods.
âž¤This imposes a strict contract on the class, ensuring consistency in structure and behavior across multiple classes.

3. Good Design Practices
âž¤Interfaces promote clear separation of responsibilities.
âž¤They lead to clean, modular, and testable code.
âž¤Developers can work independently on the interface and its implementations.

4. Loose Coupling
âž¤Classes that use interfaces depend only on method signatures, not on concrete implementations.
âž¤This allows changes in implementations without affecting dependent classes.
âž¤Enhances flexibility, extensibility, and maintainability of the system.

5. Multiple Inheritance
âž¤Java doesnâ€™t support multiple inheritance with classes, but a class can implement multiple interfaces.
âž¤This allows a class to inherit behavioral capabilities from multiple sources.

6. Supports Plug-and-Play Architecture
âž¤Interface-based design makes it easy to swap implementations (e.g., using different payment gateways, databases, or services) without changing the dependent code.

Note:
âž¤Class to Class Inheritance --> extends
âž¤Interface to Interface Inheritance --> extends
âž¤Interface to Class Inheritance --> implements

âž¤ It creates an anonymous class behind the scenes. A class without any name is called Anonymous.
âž¤Then the lambda expression creates an object and loads the method by implementing it inside the object.
âž¤Then we use an object reference to call that implemented method
âž¤Replacing instances of anonymous classes that implement functional interfaces with a concise, inline function-like syntax.

âž¤Infers the target type based on the context. â†’ It must be a functional interface.
âž¤Converts the lambda into an instance of the functional interface using invokedynamic bytecode instruction (introduced in Java 7).
âž¤Creates a synthetic method and links the lambda expression to it at runtime.

ðŸ”¹ 5. To Store and Read Values We Use: start_address + index Ã— memory_size
âž¤Internally, the array elements are stored sequentially in memory.
âž¤The memory address of any element can be calculated using:

Address = BaseÂ Address + (IndexÃ—SizeÂ ofÂ EachÂ Element)
âž¤This allows direct access to any element using its index in constant time O(1), which makes arrays efficient for random access.

ðŸ”¹ 6. Memory Allocation Happens in Sequence (Contiguous Allocation)
âž¤When an array is created, Java allocates a block of continuous memory locations to store its elements.
This contiguous allocation ensures that:
Each element is placed right after the previous one in memory.
Accessing any element is fast using index-based computation.
But the array size cannot change after creation (fixed size limitation).

Removing Duplicate elements from an array
###########################################
âž¤For the below logic array should be sorted

Steps:
1. Create a temp array which same size as that of x
int[] x = {3,3,4,5,6,6,7};
int[] temp = new int[x.length];

2. Compare i with i+1 of array x, if not equal copy the value from x[i] to temp[j] and increment j by 1

3. When you reach last index of array x, copy that value directly to temp[j];

package p1;
public class A {
	public static void main(String[] args) {
		int[] x = {3,3,4,5,6,6};
		int[] temp = new int[x.length];
		int j=0;
		for(int i=0;i<x.length-1;i++) {//i=5
			if(x[i]!=x[i+1]) {//with index 5 to 6
				temp[j] = x[i];
				j++;
			}
		}
		temp[j] = x[x.length-1];
		
		for(int arr:temp) {
			System.out.println(arr);
		}
	}
}

Finally block in Java
###########################
âž¤ This isan  extension of try try-catch block
âž¤The code that we write in the finally block will run 100%, regardless of the exception

Example:
--------
public class E {
	int x = 10;
	public static void main(String[] args) {
		try {
			int x = 100/0;
		}catch (Exception e) {
			e.printStackTrace();
		}finally {
			System.out.println("Finally");//Output: Finally
		}	
	}
}

Interview Question:
a. Difference between final, finally, and finalize
=============================================================
final
--------
-> final makes variable constant
-> final prevents overriding
-> final on class stops inheritance

finally
-------
-> This is an extension of try try-catch block
-> The code that we write in the finally block will run 100%, regardless of the exception

finalize():
----------
-> This is a method inside the Object class, and herethe  Garbage collection logic is implemented
-> We can try calling the garbage collector using System.gc(). But it will not guarantee 100% execution.
