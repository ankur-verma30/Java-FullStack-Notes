рҹ”· Agile SDLC Phases
Agile adapts the SDLC to be iterative and flexible, but the phases are still there, just continuous and overlapping:

1пёҸвғЈ Requirement Gathering & Analysis
вһӨ Done in a lightweight way.
вһӨ Requirements are captured as user stories in the Product Backlog.
вһӨ Prioritized based on business value.
вһӨ Continuous refinement via backlog grooming.

2пёҸвғЈ Planning
вһӨ Sprint Planning sets the goals for the upcoming iteration (usually 2вҖ“4 weeks).
вһӨ Team estimates tasks using story points.
вһӨ Determines what backlog items can be completed in the sprint.

3пёҸвғЈ Design(SRS - System Requirements Specification)
вһӨ Incremental design вҖ” only whatвҖҷs needed for the sprint.
вһӨ Often collaborative and flexible; evolves over time.
вһӨ Focus on simple design that supports quick iterations.

4пёҸвғЈ Development / Implementation
вһӨ Actual coding happens in short sprints.
вһӨ Frequent integrations to ensure working software.
вһӨ Practices like pair programming, TDD may be used (Extreme Programming influence).

5пёҸвғЈ Testing / Quality Assurance
вһӨ Continuous testing is integrated into development.
вһӨ Automated tests & regression tests run frequently.
вһӨ Feedback from QA team or Product Owner is immediate.

6пёҸвғЈ Review / Sprint Review
вһӨ Completed features are demonstrated to stakeholders.
вһӨ Feedback is collected to refine next sprint planning.

7пёҸвғЈ Deployment / Delivery
вһӨ Working software can be delivered at the end of each sprint.
вһӨ Supports continuous integration & continuous delivery (CI/CD).

8пёҸвғЈ Maintenance / Retrospective/ SLA( Service Line Agreement)
вһӨ Continuous improvement via Sprint Retrospective.
вһӨ Bugs, performance issues, and enhancements are added back to the backlog.

рҹ”· Key Agile SDLC Characteristics
вң”пёҸ Iterative & Incremental: SDLC is a loop of planning вҶ’ developing вҶ’ testing вҶ’ reviewing вҶ’ adapting.
вң”пёҸ Customer-centric: Continuous feedback influences every phase.
вң”пёҸ Flexible Design & Requirements: Changes donвҖҷt break the cycle; they are embraced.
вң”пёҸ Integrated Testing: No separate QA phase at the end вҖ” testing is continuous.
вң”пёҸ Rapid Delivery: Small, functional releases every sprint.

рҹ§  Core Agile Terms
1пёҸвғЈ Agile
A mindset and methodology focused on iterative development, customer collaboration, and flexibility in responding to change.

2пёҸвғЈ Iteration / Sprint
вһӨ A time-boxed period (usually 1вҖ“4 weeks) during which a specific set of work (user stories) is completed and made ready for review.

9пёҸвғЈ Development Team

1пёҸвғЈ0пёҸвғЈ Increment

1пёҸвғЈ1пёҸвғЈ Definition of Done (DoD)

1пёҸвғЈ3пёҸвғЈ Sprint Planning

1пёҸвғЈ7пёҸвғЈ Velocity

1пёҸвғЈ9пёҸвғЈ Burndown Chart

2пёҸвғЈ1пёҸвғЈ Backlog Refinement (Grooming)

2пёҸвғЈ2пёҸвғЈ Impediment / Blocker

2пёҸвғЈ4пёҸвғЈ Minimum Viable Product (MVP)

2пёҸвғЈ5пёҸвғЈ Incremental Development
Building the product in small increments, adding functionality step-by-step.

2пёҸвғЈ7пёҸвғЈ Continuous Delivery (CD)

3пёҸвғЈ0пёҸвғЈ Agile Coach

3пёҸвғЈ2пёҸвғЈ Release Planning

3пёҸвғЈ3пёҸвғЈ Acceptance Criteria

3пёҸвғЈ5пёҸвғЈ Lead Time / Cycle Time
